1. High-level Pages & Screens

Website (React / Next.js)

  • Landing / Marketing pages
  • Auth: Login / Register (Business / Worker)
  • Dashboard - Worker (Jobs feed, Applied, Accepted, Earnings)
  • Dashboard - Business (Post job, Applications, Timesheets, Payouts)
  • Jobs Listing & Search (filters: industry, role, postcode, radius)
  • Job Details & Apply
  • Profile & Documents (upload RTW, licenses)
  • Payments & Billing (view invoices, billing method)
  • Notifications center
  • Help / Support / Terms

Mobile App (React Native) — Worker & Business

  • Onboarding & Guided Profile Completion
  • Home Feed (Nearby Jobs) with map & list toggles
  • Job Details, Apply, Favourite
  • My Applications / My Jobs
  • Timesheet: Clock-in/Clock-out, Submit hours
  • Earnings & Payouts
  • Document upload (camera + gallery)
  • Push Notifications center
  • Settings & Support

2. Key Components

  • JobCard — compact job preview (title, pay, distance, start time, apply CTA)
  • Filters — role, industry, radius slider, date picker
  • Profile Completeness — progress bar, status, required documents
  • Timesheet Form — start/end, breaks, notes, evidence upload
  • Map & List View — toggle for geolocation-based browsing
  • Notifications — in-app list + push

3. Data Flow / State Management

Website: use React Context or Redux (small-to-medium) or Zustand for lightweight state. Data fetching via Axios + SWR/React Query for caching and revalidation.

Mobile: Context + React Query for API caching. Keep ephemeral UI state local to components.

6. Important Functional Notes

  • Geolocation & Radius Matching: Postcode → lat/lng on job creation and worker profile. Use Haversine for distance checks.
  • Document Upload: Use client-side compression for images, upload to S3, and store signed URLs.
  • Push Notifications: Register device tokens and send via NotificationService (queue worker).
  • Offline / Resilience: Cache last job list on device and queue failed requests (timesheet) for retry.

7. Build & Release

Website

  • Deploy Next.js on Vercel or a managed host (Cloud Run, Netlify)
  • CI: GitHub Actions (build, tests, deploy)

Mobile

  • Use EAS/Expo or bare React Native with Fastlane for automated builds
  • Implement internal beta distribution (TestFlight, Google Play Internal)
  • Prepare App Store / Play Store metadata and assets

8. Testing & QA

  • Unit tests (Jest for RN & React, PHPUnit/Pest for Laravel)
  • Integration tests for APIs
  • End-to-end (Playwright/Cypress) for web flows
  • Manual device testing matrix for major Android/iOS versions
