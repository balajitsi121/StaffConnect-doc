Category Descriptions

  • Project Setup: Initialize React Native project, configure folder structure, environment variables, and base architecture.
  • Authentication & Onboarding: Implement login, signup, OTP/email verification, and profile completion checks.
  • Navigation Setup: Configure stack, tab, drawer navigators, deep linking, and route guards.
  • Dashboard: Build staff & hotel dashboards showing metrics, counts, and shortcuts.
  • Job Posting & Management: Hotel creates jobs, edits jobs, reposts jobs, and views job status.
  • Job Feed & Radius Filtering: Staff sees available jobs filtered by location, radius, and preferences.
  • Applications Flow: Apply for jobs, check pending/accepted/rejected applications.
  • Timesheets: Staff submits timesheets, employer adjusts or rejects, resubmission flow.
  • Profiles & Document Upload: Staff uploads Right-to-Work, certificates, photos via camera/gallery.
  • Favourites: Staff favourites hotels; hotels favourite applicants.
  • Reports: Hotel views earnings, job reports with filters.
  • Push Notifications: Implement FCM, Notifee, channels, preferences, and device token registration.
  • Ratings & Reviews: Staff and hotel rating system post-shift.
  • Settings & Notification Preferences: User controls email/SMS/push notification preferences.
  • Offline Handling: Basic caching for job lists & profile data.
  • Testing: QA across iOS & Android devices and simulators.
  • App Store Deployment: Build signing, submission process for Apple App Store & Google Play Store.

Assumptions

  • Backend API will be available and documented before integration begins.
  • React Native (bare workflow or Expo with custom config) will be used.
  • Push notifications handled via Firebase (FCM) + Notifee.
  • Document uploads stored on S3 via pre-signed URLs.
  • Geolocation + radius filtering relies on backend Haversine logic.
  • Weekly payout logic handled by backend; app displays status only.
  • SMS may later use WhatsApp API if chosen by client.

Exclusions

  • Real-time chat/messaging (can be quoted separately).
  • Deep offline mode with queueing (basic caching only included).
  • Complex animations or custom maps beyond standard RN Maps.
  • Admin panel functionality (web only).

Suggested Timeline

  • Setup + navigation: 1–2 sprints
  • Core features: 4–6 sprints
  • Notifications + upload + radius logic: 2–3 sprints
  • Testing + deployment: 2 sprints

Estimated calendar duration: ~12–18 weeks based on team size.
