StaffConnect — Final Project Summary

Purpose: A future-proof, multi‑industry staffing marketplace that initially targets hotels & hospitality and is ready to expand to hospitals, events, logistics, and more.

High-level System Overview

  • Back-end API: Laravel (REST API). Single source of truth for business logic and data.
  • Website: React (Next.js recommended for SEO & performance).
  • Mobile Apps: React Native (iOS & Android).
  • Admin Panel: Laravel (Blade or Inertia/React) for verification, payouts, and configuration.
  • Payments: Stripe initially; gateway-agnostic design to add PayPal, bank transfers later.

Key Principles & Constraints

  • Design industry‑agnostic models: use Business, Worker, Job/Shift, Industry, and Skill.
  • Location-driven matching: postcode → lat/lng with radius-based filtering (Haversine formula).
  • Role-based and industry-based verification rules (pluggable configuration).
  • Single API consumed by web + mobile frontends for consistent business logic.
  • Modular services: payments, notifications, geo, document verification, and reporting are pluggable.

Core Features (Both Web & Mobile)

Workers

  • Register and upload verification documents
  • Search & apply for shifts by radius, role, and industry
  • Submit timesheets / clock-in-out (optional)
  • View earnings & payout history
  • Ratings & reviews, favourites, notifications

Businesses

  • Post jobs/shifts with postcode & search radius
  • Receive applications, shortlist, and assign staff
  • Approve or adjust timesheets and trigger payouts
  • View reports, manage jobs, messaging (optional)

Backend Modules (Laravel)

  • Auth & Roles (Sanctum) — Admin, Business, Worker
  • Industry & Skills — dynamic lists and mappings
  • Jobs/Applications — create, search, apply, assign
  • GeoLocation — postcode lookup, lat/lng storage, radius search
  • Timesheets & Payouts — approval workflow and payment orchestration
  • Documents & Verification — configurable per industry
  • Payments Service — Stripe (initial), gateway‑agnostic interface
  • Notifications Service — push (FCM/Notifee), email, SMS
  • Reports & Admin — revenue, payouts, activity, dispute handling

Backend (Laravel)

  • /app/Models — User, Business, WorkerProfile, Job, Application, Timesheet, Payment, Industry, Skill, Document
  • /app/Services — PaymentService, NotificationService, GeoLocationService, VerificationService
  • /routes/api.php — public & authenticated API routes
  • Migrations, Queues (Redis), Jobs, Notifications

Frontend (React / Next.js)

  • /pages — login, register, dashboards, jobs, job-details, profile
  • /components — JobCard, StatsCard, Navbar, Filters
  • API service layer (Axios), AuthContext, geo utils

Mobile App Structure (React Native)

  • Screens: Login, Register, WorkerDashboard, BusinessDashboard, JobList, JobDetails, Profile
  • Services: API client, notifications, geolocation
  • Use React Navigation, context/state management, and Notifee + FCM

API Endpoints (Representative)

  • POST /auth/register — register business/worker
  • POST /auth/login — login
  • GET /jobs?nearby=true — radius + role + industry filtered list
  • POST /jobs — create job (business)
  • POST /jobs/{id}/apply — apply for a job
  • POST /timesheet — submit hours
  • POST /payments/payout — trigger payout
  • GET /admin/reports — admin reports

Geo & Matching Details

  • Use postcode lookup service (UK: Postcodes.io or Google) to convert postcode → lat/lng.
  • Store lat/lng on Job and Worker records.
  • Filter using Haversine formula (SQL or application logic) to return jobs where distance ≤ job.radius.

Payments & Payouts

  • Start with Stripe Connect for marketplace flows and platform fee handling.
  • Design payment service interface to add other gateways (PayPal, bank transfer) later.
  • Payout schedule configurable (weekly default), with manual overrides via admin.

Phased Development Roadmap

  1. Phase 1 — Foundation: Laravel API, auth, roles, industry model, postcode → lat/lng support.
  2. Phase 2 — Profiles & Verification: worker/business profile flows, document upload, admin verification.
  3. Phase 3 — Jobs & Matching: job CRUD, radius-based job feed, applications.
  4. Phase 4 — Timesheets & Payments: timesheet workflow, Stripe integration, payout orchestration.
  5. Phase 5 — Frontends: React website first (worker + business), then React Native apps.
  6. Phase 6 — Admin & Reporting: dashboard, dispute management, industry settings.

Operational Considerations

  • Use queues for emails, document processing, payout jobs, and notifications.
  • Use caching (Redis) for frequently read lists (e.g., roles, skills, industry lists) and to rate-limit heavy queries.
  • Store audit logs for timesheet edits and payout overrides for compliance.
  • Prepare a data migration story to add future industries and verification types without schema breaking changes.
