Assumptions

  • Backend API endpoints for all flows are available and documented (or will be delivered 1–2 sprints ahead of frontend).
  • Postcode validation uses an external API (Postcodes.io or Google) — API keys and accounts will be provided by client or procured separately.
  • Authentication uses token-based scheme (Laravel Sanctum) and is already implemented server-side.
  • Payments are handled by backend (Stripe) — frontend only shows payment/billing views.
  • Notifications channels available: push (web push), email, SMS. Integrations (Twilio/SendGrid) are backend responsibilities; frontend implements settings & token registration UI.
  • 50% partial payout feature is NOT included as per client decision.

Exclusions (Not Included)

  • Real-time chat between Hotel and Staff (can be quoted separately).
  • Full Admin Panel (separate project scope).
  • Deep offline-first mobile behavior (basic caching for web only).
  • Complex custom analytics beyond CSV/PDF export.

Estimated calendar duration: ~10–16 weeks (depending on team size and parallel work).
